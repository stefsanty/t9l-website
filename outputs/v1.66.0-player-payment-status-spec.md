# v1.66.0 — Player payment status system (queued)

**Status:** queued; ships AFTER v1.65.x is complete. Earliest viable is post-v1.65.1 (dual-write of `position` to PlayerLeagueMembership). Cleanest is post-v1.65.2 (read-flip; `position` is canonical on PLM). Recommend waiting until at least v1.65.2 has merged + soaked before opening this branch.

**Branch name:** `claude/player-payment-status`

## Spec (verbatim from user)

```
## Objective
Implement a player payment status system for league memberships.

## Schema Changes

Add to PlayerLeagueMembership:
- paidStatus PaidStatus @default(UNPAID)
- paidAt DateTime?
- feeOverride Int? (null = use resolved fee)

New enum:
- PaidStatus { PAID UNPAID }

New model LeaguePositionFee:
- id String @id @default(cuid())
- leagueId String
- position String (e.g. "GK", "FP")
- fee Int
- Unique constraint on [leagueId, position]
- Relation to League

Add to League (or LeagueSettings if that model exists):
- defaultFee Int @default(0)
- Relation to LeaguePositionFee[]

Generate and run migration.

## Fee Resolution Logic

Create a utility resolvePlayerFee(membership: PlayerLeagueMembership & { league: League & { positionFees: LeaguePositionFee[] } }): number

Resolution order:
1. membership.feeOverride if not null
2. league.positionFees.find(p => p.position === membership.position)?.fee
3. league.defaultFee

## Admin: League Settings Page

Add a "Player Fees" section:
- Input for defaultFee
- Per-position fee rows for GK and FP (add/edit/remove)
- Save updates league + upserts LeaguePositionFee records

## Admin: Players Page (per league)

Add to each player row:
- PAID / UNPAID badge
- Effective fee display (resolved, not raw)
- Inline custom fee override input
- Toggle paid/unpaid button

Add bulk operations toolbar (appears when rows selected):
- Select all checkbox
- "Mark Paid" / "Mark Unpaid" buttons
- Sets paidStatus and paidAt (null if unpaid) on all selected memberships

Paid date (paidAt) visible in player detail/expanded row, admin only.

## Player-Facing Banner

On all pages scoped to a specific league (schedule, roster, standings, etc.) for authenticated users:

Check if the current user has a PlayerLeagueMembership for that league with paidStatus === UNPAID.

If true, show a permanent (non-dismissible) banner:
"You have not paid your ¥{effectiveFee} league fee."

Banner should be consistent across all league-scoped pages. Compute effective fee server-side using resolvePlayerFee.

## Server Actions / API Routes

- updateLeagueFeeSettings(leagueId, defaultFee, positionFees[])
- updateMembershipPaidStatus(membershipId, status) — sets paidAt timestamp
- bulkUpdatePaidStatus(membershipIds[], status)
- updateMembershipFeeOverride(membershipId, feeOverride: number | null)

All mutations: verify caller is admin. No IDOR — validate that the target memberships belong to the specified league before updating.

## Audit
- Verify resolvePlayerFee is used everywhere fee is displayed — never raw defaultFee
- Verify banner appears on all league-scoped pages, not just one
- Verify bulk action validates membership→league ownership before update
```

## Coordination notes (added by Claude)

- **Position field source.** Spec keys fee resolution off `membership.position`. v1.65.0 added `PlayerLeagueMembership.position` (per-league, nullable). Through v1.65.1 (dual-write) it mirrors `Player.position`; from v1.65.2+ it's canonical. v1.66.0 should read `membership.position` (the new canonical) once v1.65.2 has merged + soaked. If shipped between v1.65.1 and v1.65.2, fall back to `Player.position` defensively.
- **GK/FP position taxonomy.** Spec says `position: String (e.g. "GK", "FP")`. The current schema has `PlayerPosition` enum `GK | DF | MF | FW`. "FP" (field player) is not in the enum. Two options: (a) treat `LeaguePositionFee.position` as a free string with admins inputting "GK" / "FP" / "DF" / "MF" / "FW" / etc. and matching against Player position via lookup; (b) add a new enum or extend `PlayerPosition`. Recommend (a) since fees are per-league and admins may want richer categorization (Captain / Reserve / etc.). Document the GK + FP convention in the admin UI.
- **Banner placement.** "League-scoped pages" are: apex `/`, `/id/[slug]`, `/id/[slug]/md/[id]`, `/schedule`, `/stats`. Render the banner via the unified `Dashboard` component plus the schedule/stats wrappers — same pattern as v1.25.0's apex+subdomain unification.
- **`paidAt` semantics.** `null` when `paidStatus === UNPAID`; set to `now()` when admin flips to PAID. Re-flipping to UNPAID nulls it again.
